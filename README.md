# Audio--text
**Название**: конвертер видео/аудиофайлов в текст.<br>
**Описание**: приложение предназначено для конвертации аудио/видеофайлов в текст или текстовый файл.<br> Затем пользователь может скачать файл или просто скопировать текст.<br>
**Целевая аудитория**: блогеры, видеомейкеры, копирайтеры.<br>
**Ключевой функционал**: загрузка видео/аудиофайлов, конвертация видео/аудиофайлов в текст или текстовый файл, скачивание текстового файла или копирование текста напрямую.

## Структура приложения:
**Главная** (/) — Основной интерфейс: загрузка файлов, выбор языка, запуск транскрибации.<br>
**Личный кабинет** (/dashboard) — Список всех прошлых расшифровок пользователя, возможность скачать текст повторно или отредактировать его.<br>
**Тарифы и поддержка** (/billing) — Здесь пользователь выбирает пакеты токенов для API или просто благодарит автора.<br>
**Аутентификация** (/auth) — Вход, регистрация и управление профилем.<br>
**Взаимодействие с API** (/api) — Управление токенами и статистика вызовов.<br>
**Руководство по API** (/docs) — Интерактивная документация (Swagger/Redoc) по эндпоинтам.<br>
**Блог** (/blog) — Статьи о том, как использовать транскрибацию в разных сферах (*для журналистов, студентов и т. д.*).<br>
**О разработчике** (/about) — Ваше портфолио и другие проекты.<br>

## Технологический стек: 
**Frontend**: Vue.js, Vue Router 4, Pinia.<br>
**Backend**: Python, Flask, SQLite.<br>
**Инфраструктура**: frontend на Vercel, backend и БД локально, нейросеть на Modal.<br>

## Модели данных:
Сущность: **User** - основные поля: ID, username, email, password_hash, show_add, api_tier.<br>
Сущность: **API_tier** - основные поля: id, name, token_limit.<br>
Сущность: **User_Tokens** - основные поля: user_id, balance, last_replenishment_at.<br>

## Ключевые эндпоинты API:
**Регистрация**: <br>
**Метод**: POST.<br>
**Путь**: /auth/register.<br>
**Описание**: создает новый аккаунт пользователя.<br>
**Body(JSON)**: {"email":"..." , "password":"...", "username":"..."}.<br>
**Ответ**: 201 Created и данные созданного пользователя(*без пароля*).<br>

**Вход**:
**Метод**: POST.<br>
**Путь**: /auth/login<br>
**Описание**: проверяет учетные данные и разрешает доступ.<br>
**Body(JSON)**: {"email":"..." , "password":"..."}<br>
**Ответ**: 200 OK и id пользвателя.<br>

**Получение количества оставшихся токенов**:
**Метод**: GET.<br>
**Путь**: /api/token.<br>
**Описание**: возвращает количество осташихся токенов.<br>
**Body(JSON)**: {"id":"..."}<br>
**Ответ**: 200 OK и количество токенов.<br>
